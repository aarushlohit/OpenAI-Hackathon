# Hermes-X Blueprint

Mission: protect students from internship and recruitment scams through autonomous cyber intelligence.

Phase 1 deliverables:

- Async terminal investigation command.
- Structured event stream.
- AI gateway abstraction for text and vision providers.
- Automatic failover routing skeleton.
- Persistent memory interface.
- Security-first configuration.
- Professional operator documentation.

Phase 2 will add real provider integrations, sandboxed artifact analysis, database adapters, and domain-specific scam agents.

Phase 2+ foundation now includes modality routes for text, vision, audio, and embeddings, with failover, circuit breaker state, and provider metrics centralized in the gateway.

Phase 3 adds the first autonomous cyber intelligence layer: behavior analysis, OSINT correlation, OCR-first vision analysis, realtime agent streaming, and explainable threat scoring.

Later phases will add the Flutter operator interface, collaborative case management, graph intelligence, and institutional reporting workflows.

Phase 5 adds the cyber intelligence experience layer: websocket-native event delivery, replay mode, visual timeline foundations, graph projection contracts, and a Flutter client that remains presentation-only.

Phase 7 adds autonomous operations: continuous threat feed, escalation rules, demo cinema mode, storytelling, executive reports, replay integrity, and SOC-grade presentation surfaces.

Phase 8 adds governed production hardening: schema registry, event versioning, replay determinism, agent contracts, evaluation, deployment, audit mode, archival, and tool registry foundations.
